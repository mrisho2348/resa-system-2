DataTable Template Usage Guide

Overview

The DataTable template provides a reusable, feature-rich table component with search, pagination, export options, and responsive design.

Basic Usage

Step 1: Extend the base template
{% verbatim %}{% extends "pembaadmin_template/base_template.html" %}
{% load static %}{% endverbatim %}
Step 2: Include the DataTable template
{% verbatim %}{% extends "pembaadmin_template/datatable_template.html" %}{% endverbatim %}

Required Blocks

Block Name Purpose Required
table_title Title displayed in the card header Yes
table_id Unique ID for the table (used in JavaScript) Yes
table_headers Table header row content Yes
table_body Table body content (data rows) Yes
table_actions Action buttons in card header Optional
table_id_js Table ID for JavaScript (usually same as table_id) Yes

Best Practices

Do's
  • Use unique table IDs for each table
  • Include action columns for CRUD operations
  • Add proper error handling for empty states
  • Use appropriate badges for status indicators
  • Implement responsive design considerations
Don'ts
  • Don't forget to handle empty data states
  • Avoid too many columns on mobile
  • Don't use generic table IDs
  • Avoid complex nested data in cells
  • Don't forget to test export functionality
